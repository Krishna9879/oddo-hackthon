OPTION 1: Quick Fix (If Database Already Imported)
---------------------------------------------------
1. Open phpMyAdmin
2. Select 'workzen' database
3. Click on 'SQL' tab
4. Open file: database/fix-passwords.sql
5. Copy all content and paste in SQL tab
6. Click 'Go'
7. Done! All passwords are now fixed!

OPTION 2: Fresh Import (Recommended)
--------------------------------------
1. Open phpMyAdmin (http://localhost/phpmyadmin)
2. If 'workzen' database exists, DELETE it first
3. Click on "New" in the left sidebar
4. Enter database name: workzen
5. Select Collation: utf8mb4_general_ci
6. Click "Create"

STEP 2: Import Schema (Option 2 Only)
--------------------------------------
1. Select the "workzen" database from the left sidebar
2. Click on the "Import" tab at the top
3. Click "Choose File" button
4. Navigate to: database/schema.sql
5. Click "Go" button at the bottom
6. Wait for success message

STEP 3: Import Sample Data (Option 2 Only)
-------------------------------------------
1. Stay in the "workzen" database
2. Click on the "Import" tab again
3. Click "Choose File" button
4. Navigate to: database/sample-data.sql
5. Click "Go" button at the bottom
6. Wait for success message

========================================
TROUBLESHOOTING
========================================

If you get errors:
1. Make sure database is empty before importing schema.sql
2. Import schema.sql FIRST, then sample-data.sql
3. Check if MySQL server is running
4. Verify .env.local has correct database credentials
5. Make sure database name is exactly "workzen"
